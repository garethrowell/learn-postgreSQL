
1. Introduction to Postgresql

PL/PgSQL is procedural language

ACID ~ atomicity, consistency, isolation, durability
atomicity ~ complex operations processed as single instruction
consistency ~ data always kept consistent and not corrupt by partially
performed operations
isolation ~ databse handles concurrency without corruption from 
interleaved changes
durability ~ db engine protects data as much as possible during 
software or hardware failure

PGDATA ~ directory where data is stored
WALS ~ write-ahead logs and data storage intent written ahead of 
data changes#
catalogs ~ info on status of all user defined tables
postmaster ~ single process to bootstrap instance
server ~ serves up your data to applications and users
client ~ client tool to connect to database server
contrib ~ extensions and utilities from the PGDG
PL/Python ~ allows Python directly within server
